## 1. 前置条件

### 1.1 开发环境

- **Android Studio**：建议使用 2022.3+ (Hedgehog) 或更新版本
  - 下载地址：https://developer.android.com/studio
- **JDK 17**：Android Studio 自带，无需单独安装
- **Android SDK**：API Level 33+（Android 13）
  - 通过 Android Studio → SDK Manager 安装

### 1.2 手机准备

1. **开启开发者选项**：
   - 进入「设置 → 关于手机」
   - 连续点击「软件版本号」7 次，直到提示"已开启开发者选项"

2. **开启 USB 调试**：
   - 进入「设置 → 更多设置 → 开发者选项」
   - 打开「USB 调试」开关
   - （Vivo）同时打开「USB 安装」开关

3. **使用数据线连接电脑**：
   - 手机弹出授权提示时，选择「允许」并勾选「始终允许」
   - 确认手机屏幕顶部状态栏出现 USB 调试图标

4. **验证连接**（在电脑终端执行）：
   ```bash
   adb devices
   ```
   应显示类似如下内容：
   ```
   List of devices attached
   XXXXXXXXXX    device
   ```
   如果显示 `unauthorized`，请在手机上重新确认授权。

## 2. 构建项目

### 方式一：使用 Android Studio（推荐）

1. 打开 Android Studio
2. 选择 `File → Open`，导航到 `/root/xiaobo-agent/android-app/` 目录
3. 等待 Gradle 同步完成（首次可能需要 5-10 分钟下载依赖）
4. 同步完成后，菜单栏选择 `Build → Build Bundle(s) / APK(s) → Build APK(s)`
5. 等待构建完成，右下角会弹出通知，点击 `locate` 可以找到生成的 APK 文件

### 方式二：使用命令行

在项目根目录（`android-app/`）下执行：

```bash
# 给 gradlew 添加执行权限
chmod +x gradlew

# 构建 Debug 版本 APK
./gradlew assembleDebug
```

构建完成后，APK 文件位于：
```
app/build/outputs/apk/debug/app-debug.apk
```

## 3. 安装 APK

### 方式一：通过 ADB 安装（推荐）

```bash
adb install app/build/outputs/apk/debug/app-debug.apk
```

安装成功后会输出 `Success` 提示。

如果需要覆盖安装（更新已有版本）：
```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 方式二：通过文件传输安装

1. 将 APK 文件传到手机（微信/QQ 文件传输、数据线拷贝等）
2. 在手机上打开「文件管理」，找到 APK 文件
3. 点击安装（Vivo 可能提示"安装未知来源应用"，需要在弹出的提示中允许）

### 方式三：通过 Android Studio 直接运行

1. 在 Android Studio 中确认手机已连接（右上角设备列表可看到）
2. 点击绿色三角形 `Run` 按钮
3. 选择目标设备，等待安装并自动启动

## 4. Vivo 手机权限配置

本应用需要以下特殊权限才能正常采集使用数据。**必须逐项开启，否则应用无法正常工作。**

### 4.1 使用情况访问权限（UsageStats）

这是**最关键**的权限，用于读取各个应用的使用时长。

**操作步骤：**

1. 打开手机「设置」
2. 进入「安全与隐私 → 权限管理」（部分 Vivo 机型路径为「设置 → 更多设置 → 权限管理」）
3. 点击右上角「…」或「更多」，选择「使用情况访问权限」
   - 或直接搜索"使用情况"
4. 在应用列表中找到「小波手机监控」（或显示为包名 `com.xiaobo.phone`）
5. 点击进入，选择「允许」

> ⚠️ **注意**：部分 Vivo 机型的路径为：
> - 「设置 → 应用与权限 → 权限管理 → 使用情况访问权限」
> - 「设置 → 安全与隐私 → 特殊应用权限 → 使用情况访问权限」
>
> 如果找不到，可以在「设置」顶部搜索栏中搜索"使用情况"直接定位。

### 4.2 其他需要开启的权限

在手机「设置 → 应用管理 → 找到小波手机监控」中：

| 权限 | 操作 |
|------|------|
| 通知权限 | 设为「允许」（应用通过前台通知保持后台运行） |
| 自启动 | 打开开关（手机重启后自动启动） |
| 关联启动 | 打开开关 |
| 省电策略 | 设为「无限制」 |

> 💡 部分权限可以在应用内点击按钮后自动跳转到对应设置页面，按照提示操作即可。

## 5. Vivo 后台保活设置

Vivo 系统对后台应用管控严格，**如果不做以下设置，应用可能在锁屏后被系统杀死**，导致数据无法正常采集和上报。

### 5.1 允许后台运行

1. 进入「设置 → 应用管理 → 小波手机监控」
2. 找到「耗电保护」或「电池」选项
3. 选择「无限制运行」/「允许后台高耗电」

### 5.2 关闭电池优化

1. 进入「设置 → 电池 → 后台耗电管理」（或「更多设置 → 电池」）
2. 找到「小波手机监控」
3. 选择「无限制」或「允许后台运行」

### 5.3 加入应用白名单（重要）

1. 进入「设置 → 更多设置 → 应用管理 → 小波手机监控」
2. 打开「允许后台活动」或「后台弹出界面」

### 5.4 锁定应用（从最近任务中）

1. 打开应用后，按「多任务键」（或底部上滑停顿）
2. 在最近任务列表中找到「小波手机监控」
3. **向下拉卡片**（或点击锁定图标 🔒），使其出现锁定标记
4. 这样清理后台时不会误杀该应用

### 5.5 开机自启动

1. 进入「设置 → 应用管理 → 小波手机监控」
2. 打开「自启动」开关
3. 确保手机重启后应用能自动恢复运行

## 7. 常见问题排查

### Q1：adb devices 显示 no devices / offline

- 确认数据线支持数据传输（非纯充电线）
- 重新插拔数据线
- 在手机上确认 USB 调试授权弹窗
- 尝试更换 USB 接口
- Vivo 手机请确认「开发者选项 → USB 安装」已打开

### Q2：构建时报错 "Could not resolve" / 下载超时

- 检查网络连接（Gradle 依赖需要从 Maven 仓库下载）
- 如果在国内网络，可在 `gradle.properties` 中配置镜像源：
  ```properties
  mavenRepositoryUrl=https://maven.aliyun.com/repository/public
  ```
- 或在 `build.gradle` 中配置阿里云镜像

### Q3：应用安装后提示"无法获取使用数据"

- **未开启使用情况访问权限**：参考 [第 4.1 节](#41-使用情况访问权限usagestats)
- 部分 Vivo 机型需要额外开启「辅助功能」权限

### Q4：应用在后台被杀死，无法持续运行

- **未做后台保活设置**：参考 [第 5 节](#5-vivo-后台保活设置)
- 检查是否开启了电池优化中的「无限制」选项
- 确认已将应用锁定在最近任务中
- Vivo OriginOS/FuntouchOS 较新版本可能需要额外在「i管家 → 应用管理」中设置

### Q5：数据上报失败

- 检查手机网络是否正常
- 确认服务器地址可达：在手机浏览器访问 `http://1.117.61.172:8088`
- 检查是否有防火墙/VPN 拦截了网络请求
- 查看应用日志：`adb logcat -s XiaomiAgent` 或 `adb logcat | grep xiaobo`

### Q6：安装 APK 时提示"禁止安装未知来源应用"

1. 点击提示中的「设置」按钮
2. 在弹出页面中打开「允许安装未知来源应用」
3. 返回重新安装

或通过 ADB 安装可绕过此限制：
```bash
adb install -r app-debug.apk
```

### Q7：如何查看应用运行日志

```bash
# 查看应用相关日志
adb logcat | grep -i "xiaobo\|phone.*stats\|UsageStats"

# 过滤应用包名
adb logcat --pid=$(adb shell pidof com.xiaobo.phone)
```

### Q8：如何完全卸载应用

```bash
adb uninstall com.xiaobo.phone
```

或在手机上：「设置 → 应用管理 → 小波手机监控 → 卸载」

## 技术架构简述

```
android-app/
├── app/
│   ├── src/main/
│   │   ├── java/com/xiaobo/phone/   # Kotlin 源码
│   │   ├── res/                      # 资源文件
│   │   └── AndroidManifest.xml       # 应用清单
│   └── build.gradle                  # 模块构建配置
├── build.gradle                      # 项目构建配置
├── gradle/                           # Gradle Wrapper
└── README.md                         # 本文档
```

- **语言**：Kotlin
- **最低 SDK**：26 (Android 8.0)
- **目标 SDK**：34 (Android 14)
- **采集方式**：通过 `UsageStatsManager` API 读取应用使用时长
- **上报方式**：HTTP POST 请求，JSON 格式，每 30 分钟执行一次
- **后台机制**：前台服务 + 通知栏常驻通知 + WorkManager 定时任务
